# `T4.2.md`

## Goal

Finish PQ integration and make batch verification production-ready:

* Concrete ML-DSA-44 adapter (`Pq44`) implementing our `SignatureScheme`.
* Correct generic bounds + type usage in `consensus_sig`.
* Batch verification path (`verify_many`) with Rayon (optional) and serial fallback.
* Tests un-ignored behind feature flag and all passing.
* Proof bundle generation for audit.

## Key fixes & upgrades

1. **Adapter & API correctness**

   * `crates/ledger/src/pq44_runtime.rs` implements:

     * `SignatureScheme for Pq44` with
       `PublicKey = pqcrypto_mldsa::mldsa44::PublicKey`,
       `Signature = pqcrypto_mldsa::mldsa44::DetachedSignature`.
     * Uses `pqcrypto_mldsa::mldsa44::verify_detached_signature(sig, msg, pk).is_ok()`.

2. **Generic constraints & types**

   * `consensus_sig::verify_core::<Pq44>(...)` and `verify_many::<Pq44>(...)` constrained to the concrete ML-DSA types.
   * `chain_id` passed as slice in `verify_core` (`&[u8]`), kept as `[u8; 20]` in callers; borrow where needed.
   * `impl CertLookup + Sync + ?Sized` used so `&dyn CertLookup` callers compile.

3. **Batch verification path**

   * `verify_many`:

     * Rayon path under `--features rayon` (`par_iter()`), serial fallback via `iter()`.
     * Collects `Vec<bool>` then converts to `BitVec` for compatibility.

4. **Tests**

   * Added conditional un-ignore:

     ```rust
     #[cfg_attr(not(feature = "consensus-tests"), ignore)]
     ```
   * All tests green with:

     * `--features "pq44-runtime,rayon,metrics,consensus-tests"`.

5. **Cargo / features hygiene**

   * No duplicate deps (e.g., `zeroize`) and correct versions:

     * `pqcrypto-mldsa = "0.1"`, `pqcrypto-traits = "0.3"`.
   * `pq44-runtime` gates all imports that mention `mldsa44`.

6. **Benches**

   * Criterion benches stabilized; tip to use `-- --measurement-time 10`.
   * Performance numbers recorded in proof.
